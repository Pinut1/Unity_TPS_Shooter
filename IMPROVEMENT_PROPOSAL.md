# Pinut1/Unity_TPS_Shooter 개선 제안서

본 제안서는 프로젝트의 사용성, 확장성, 그리고 AI 에이전트와의 상호작용(AI Interaction)을 강화하기 위한 5가지 고영향력 개선 사항을 기술합니다.

## 1. 입력 시스템 추상화 (Input System Abstraction)
**문제점**: 현재 코드는 `Input.GetMouseButton(0)`과 같이 하드코딩된 입력을 사용합니다. 이는 키 변경(Remapping)을 어렵게 하고, 외부 AI 에이전트가 게임을 제어하기 힘들게 만듭니다.
**제안**: `InputManager` 클래스를 도입하여 입력을 추상화합니다.
**효과**:
- **사용성**: 플레이어가 키 설정을 변경할 수 있습니다.
- **AI 상호작용**: AI 에이전트가 `InputManager`를 통해 가상 입력을 주입(Inject)할 수 있어 강화학습(RL) 환경 구축이 용이해집니다.

## 2. 모듈형 AI 아키텍처 (Modular AI Architecture)
**문제점**: `Zombie.cs` 등의 적 AI가 단순한 `if-else` 구조로 되어 있어, 복잡한 행동 패턴을 구현하거나 확장하기 어렵습니다.
**제안**: 유한 상태 머신(FSM) 또는 행동 트리(Behavior Tree) 패턴을 도입합니다.
**효과**:
- **확장성**: 새로운 적 유형이나 행동 패턴(예: 엄폐, 협동 공격)을 쉽게 추가할 수 있습니다.
- **사용성**: 더 지능적이고 예측 불가능한 적을 통해 게임의 재미를 높입니다.

## 3. 이벤트 기반 UI 시스템 (Event-Driven UI System)
**문제점**: UI 로직과 게임 로직이 강하게 결합되어 있어, 하나를 수정하면 다른 하나가 깨지기 쉽습니다 (예: `WeaponSystem` 내의 UI 코드).
**제안**: 옵저버 패턴(Observer Pattern)을 활용하여 데이터 변경 시 이벤트를 발생시키고, UI가 이를 구독(Subscribe)하도록 변경합니다.
**효과**:
- **확장성**: UI 디자인을 변경하거나 새로운 HUD 요소를 추가할 때 게임 로직을 건드릴 필요가 없습니다.
- **성능**: 매 프레임 UI를 갱신하는 비효율을 제거합니다.

## 4. 전역 게임 매니저 및 서비스 로케이터 (Global Game Manager & Service Locator)
**문제점**: `ObjectPool`, `InputManager`, `SoundManager` 등 주요 시스템에 대한 접근이 파편화되어 있습니다.
**제안**: `GameManager`를 도입하여 게임의 생명주기(시작, 일시정지, 종료)를 관리하고, 서비스 로케이터 패턴으로 주요 시스템에 대한 전역 접근을 제공합니다.
**효과**:
- **확장성**: 새로운 시스템(예: 퀘스트 시스템)을 추가하고 통합하기 쉬워집니다.
- **AI 상호작용**: AI 에이전트가 게임 상태(점수, 종료 여부 등)를 파악하기 쉬운 중앙화된 인터페이스를 제공합니다.

## 5. AI 에이전트용 관측/제어 API (Observation/Control API for AI Agents)
**문제점**: 외부 AI(예: Python 기반의 강화학습 에이전트)가 게임 상태를 읽거나 명령을 내리기 위한 표준화된 방법이 없습니다.
**제안**: 게임 내 주요 변수(플레이어 체력, 탄약, 적 위치 등)를 구조화된 데이터로 반환하고, 외부 명령을 수행할 수 있는 `IAgentInterface`를 정의합니다.
**효과**:
- **AI 상호작용**: Unity ML-Agents와 같은 도구와 쉽게 연동할 수 있어, 자율 주행 캐릭터나 봇 개발이 가능해집니다.
- **확장성**: 게임을 테스트하는 자동화 봇을 만들기 쉬워집니다.
